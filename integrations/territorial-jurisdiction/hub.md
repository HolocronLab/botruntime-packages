# Подсудность.РФ

Action-only интеграция для API [территориальная-подсудность.рф](https://территориальная-подсудность.рф/api).
Определяет районный или городской суд и участок мирового судьи по адресу здания либо координатам.

## Конфигурация

| Поле | Тип | Назначение |
| --- | --- | --- |
| `apiToken` | string, secret | Персональный токен из личного кабинета сервиса. |

Токен хранится в per-install configuration как secret. API поставщика требует передавать его в query-параметре;
интеграция не выводит URL запросов, токен и адреса в логи или тексты ошибок.

При подключении интеграция вызывает `GET /v1/account`: невалидный или заблокированный токен обнаруживается до первого действия.

## Actions

- `findByAddress({ address })` — поиск по адресу вида «Москва, Тверская, 1». Передавайте адрес с точностью до здания,
  без квартиры, помещения, этажа и других уточнений внутри здания.
- `findByCoordinates({ latitude, longitude })` — поиск по числовым координатам. Диапазоны проверяются до запроса.
- `getAccount({})` — тариф, баланс и остаток дневного лимита.

Для бесплатного тарифа `balance` равен `null`: поставщик ведёт только дневной лимит запросов.

Поиск возвращает:

- `districtCourt` — районный или городской суд;
- `magistrateCourt` — участок мирового судьи;
- `remaining` — оставшиеся запросы на день для тарифа `free` либо остаток в рублях для `balance`;
- распознанные адрес и координаты.

Если один из судов не найден в базе поставщика, соответствующее поле равно `null`. Поля `site`, `email` и `tel`
могут отсутствовать. `site` и `email` иногда содержат несколько значений через пробел — так их возвращает API.

## Надёжность

- Используется актуальный endpoint `/v1/`, а не legacy `/api/`.
- Таймаут одного HTTP-вызова — 15 секунд.
- Сетевые ошибки, HTTP 429/5xx и временные ошибки геокодера повторяются до трёх раз с экспоненциальной задержкой.
- Бизнес-ошибки поставщика приходят с HTTP 200 и `data.status = 0`; интеграция преобразует их в fail-loud RuntimeError.
- Ответ ограничен 1 МиБ и проверяется перед возвратом в бота.

По текущим условиям поставщика бесплатный тариф даёт 50 запросов в сутки, платный списывает 1 ₽ за запрос.
Лимиты и цены контролирует внешний сервис и может менять независимо от этой интеграции.
